# Phase 2: MedSAM2 3D Segmentation Agent — Workflow Documentation

## Overview

Phase 2 extends Phase 1 (2D bbox annotation) into full 3D segmentation. The agent:

1. Annotates bounding boxes on 10 sampled MRI slices (Phase 1 capability)
2. Seeds MedSAM2 from the key (largest cross-section) slice and propagates the mask through the entire volume
3. Refines low-quality slices using point prompts (foreground/background corrections)

The model is trained via supervised fine-tuning (SFT) on agent trajectories in ms-swift's hermes format, using Qwen3-VL-8B-Instruct as the base model with LoRA.

## Tool Schema

Four tools are defined for Phase 2:

```python
TOOLS = [
    {
        "name": "add_bbox",
        "description": "Annotate the 2D bounding box of a lesion on the specified Z slice.",
        "parameters": {
            "z_index": int,          # Z-axis index of the target slice
            "bbox": [x1, y1, x2, y2] # pixel coordinates, top-left to bottom-right
        }
    },
    {
        "name": "run_medsam2",
        "description": "Trigger MedSAM2 3D segmentation seeded from a single key slice. "
                       "Propagates forward and backward through the entire volume.",
        "parameters": {
            "key_z": int,            # Z index of the key (largest cross-section) slice
            "bbox": [x1, y1, x2, y2] # bbox on the key slice to seed MedSAM2
        }
    },
    {
        "name": "add_point",
        "description": "Refine the mask on a specific Z slice using point prompts.",
        "parameters": {
            "z_index": int,          # slice to refine
            "points": [[x, y], ...], # pixel coordinates
            "labels": [1/0, ...]     # 1=foreground, 0=background
        }
    },
    {
        "name": "finish_3d_segmentation",
        "description": "Confirm 3D segmentation is complete. Terminates the task.",
        "parameters": {}
    }
]
```

**Tool evolution across phases:**

| Phase | Tools |
|---|---|
| Phase 1 | `add_bbox`, `finish_bbox_annotation` |
| Phase 2 | `add_bbox`, `run_medsam2`, `add_point`, `finish_3d_segmentation` |
| Phase 3 | `add_bbox`, `run_medsam2`, `add_point`, `scroll`, `get_slice`, `finish_3d_segmentation` |

## Trajectory Structure (15 turns)

```
Turn 1   [user]            Task instruction + 10 Z-indexed MRI slices
Turn 2   [assistant]       Global spatial analysis CoT (no tool calls)
Turn 3   [tool_call × N]   Parallel add_bbox, one per lesion-containing slice
Turn 4   [tool_resp × N]   Rendered bbox image + IoU per slice
Turn 5   [assistant]       Bbox review CoT (pass/fail per slice)
Turn 6   [tool_call × M]   Correction add_bbox for slices with IoU < 0.75
Turn 7   [tool_resp × M]   Corrected bbox renders (present only when M > 0)
Turn 8   [assistant]       "Verified. Initiating 3D segmentation from key slice Z=K …"
Turn 9   [tool_call]       run_medsam2(key_z=K, bbox=[...])
Turn 10  [tool_response]   Propagation results: all N sampled slices with mask images + Dice
Turn 11  [assistant]       Mask review CoT (flag slices with Dice < 0.70)
Turn 12  [tool_call × P]   Parallel add_point for poor slices (P ≤ N)
Turn 13  [tool_resp × P]   Corrected mask image + Dice per refined slice
Turn 14  [assistant]       "All slices meet quality threshold. Segmentation complete."
Turn 15  [tool_call]       finish_3d_segmentation()
```

### Turn 2 — Spatial Analysis CoT

Required fields (generated by `generate_spatial_cot`):
- Lesion Z range and how many sampled slices contain it
- Key slice Z index (largest cross-sectional area)
- Morphology trend (peak location: top / middle / bottom)
- Explicit no-lesion declaration for empty slices

### Turn 10 — run_medsam2 Tool Response

Packs all N mask images into one response to satisfy ms-swift's 1:1 tool_call ↔ tool_response pairing:

```json
{
  "status": "propagation_complete",
  "key_z": 38,
  "slices": [
    {"z_index": 15, "mask_image": "<image>", "dice_with_gt": 0.84},
    {"z_index": 19, "mask_image": "<image>", "dice_with_gt": 0.95},
    ...
  ]
}
```

### Loss Computation Scope

| Content | Loss computed? |
|---|---|
| Spatial analysis CoT (assistant) | Yes |
| `add_bbox` tool_calls | Yes |
| Bbox review CoT (assistant) | Yes |
| Correction `add_bbox` tool_calls | Yes |
| `run_medsam2` tool_call | Yes |
| Mask review CoT (assistant) | Yes |
| `add_point` tool_calls | Yes |
| `finish_3d_segmentation` tool_call | Yes |
| All `tool_response` messages | **No** |
| `user` messages (instructions + images) | **No** |

## Dataset Generation

### Source Data

### Script: `convert_to_agent_trajectory_phase2.py`

**Key constants:**

| Constant | Value | Description |
|---|---|---|
| `N_SLICES` | 10 | Slices sampled per volume |
| `JITTER_RATIO` | 0.05 | Random bbox jitter (5% of bbox size) |
| `FAILURE_INJECT_RATIO` | 0.10 | Fraction of samples with a deliberately bad bbox |
| `FAILURE_OFFSET_RATIO` | 0.30 | Offset magnitude for injected failures |
| `IOU_PASS_THRESHOLD` | 0.75 | Bbox IoU gate; below triggers correction |
| `DICE_PASS_THRESHOLD` | 0.70 | Mask Dice gate; below triggers add_point |
| `MIN_LESION_FRAMES` | 3 | Minimum frames with lesion; else sample skipped |
| `N_FG_POINTS` | 3 | Foreground correction points per slice |
| `N_BG_POINTS` | 2 | Background correction points per slice |
| `RENDER_SIZE` | 256 | Render image size (px); reduces visual tokens |

### Core Functions

#### `build_ordered_annos(meta, mask_dict) → list[dict]`
Iterates all videos and expressions in sorted order, pairing each with its RLE masks and frame list. Returns a flat list of annotation dicts ready for processing.

#### `sample_slices(non_none_z, n=10) → list[int]`
Uniformly samples `n` Z indices spanning the lesion's Z range using `np.linspace`. All sampled indices are guaranteed to be within the lesion volume boundary.

#### `generate_spatial_cot(sampled, masks, caption) → (str, int)`
Generates the Turn 2 Chain-of-Thought text and identifies `key_z` (slice with maximum GT mask area). Includes trend analysis based on where the peak area falls:
- Peak near centre (±1 from middle) → "near-symmetric ellipsoid"
- Peak in upper half → "lesion skewed toward upper slices"
- Peak in lower half → "largest at the bottom, shrinking upward"

#### `add_jitter(bbox, img_w, img_h, jitter_ratio=0.05) → list[int]`
Adds uniform random noise to each bbox coordinate, scaled by `jitter_ratio × bbox_size`. Coordinates are clipped to image bounds and `x1 < x2`, `y1 < y2` is enforced.

#### `inject_failure(bbox, img_w, img_h, offset_ratio=0.30) → list[int]`
Shifts a bbox by `offset_ratio × size` to simulate a badly localised prediction (IoU < 0.6 with GT). Applied to 10% of samples on one randomly chosen slice.

#### `compute_iou(bbox, gt_2d) → float`
Computes IoU between a predicted bbox and the bounding rectangle of a GT binary mask. Returns 1.0 if GT is empty (no lesion — annotation is trivially correct).

#### `dice_score(pred, gt) → float`
Pixel-level Dice coefficient between two boolean masks: `2 × |pred ∩ gt| / (|pred| + |gt|)`. Returns 1.0 when both masks are empty.

#### `sample_correction_points(pred_mask, gt_mask, n_fg=3, n_bg=2) → (list, list)`
Generates point prompts from segmentation error regions:
- **Foreground points** (`label=1`): sampled from `GT=1 AND pred=0` (false negatives — missed lesion)
- **Background points** (`label=0`): sampled from `pred=1 AND GT=0` (false positives — over-segmentation)

Returns `(points, labels)` where `points` is `[[x, y], ...]`.

#### `render_bbox_overlay(img_path, bbox, save_path)`
Draws a red rectangle on the source JPEG and saves at `RENDER_SIZE × RENDER_SIZE` (256×256). Annotation is drawn at original resolution before downscaling to preserve coordinate accuracy.

#### `render_mask_overlay(img_path, mask, save_path, points=None, labels=None)`
Overlays a semi-transparent green mask on the source JPEG. Optionally draws correction points (red = foreground, blue = background). Saved at 256×256.

#### `load_and_preprocess_volume(jpeg_dir, device) → (tensor, H, W)`
Loads all JPEG slices from a case directory, normalises to `[0,1]` float, converts to 3-channel, pads/resizes to `(D, 3, 512, 512)`, and moves to the target device. Returns the tensor and original `(H, W)` for bbox scaling.

#### `run_medsam2_propagation(predictor, img_tensor, orig_H, orig_W, key_z, bbox_512, device) → dict[int, ndarray]`
Full bidirectional MedSAM2 propagation in three phases:

```
Phase A (key slice):
  state ← init_state(img_tensor)
  add_new_points_or_box(state, frame_idx=key_z, box=bbox_512)
  key_mask ← logits > 0  (uint8)
  reset_state(state)

Phase B (forward):
  state ← init_state(img_tensor)
  add_new_mask(state, frame_idx=key_z, mask=key_mask)
  for each frame in propagate_in_video(start=key_z, reverse=False):
      result[frame] = logits > 0

Phase C (backward):
  state ← init_state(img_tensor)
  add_new_mask(state, frame_idx=key_z, mask=key_mask)
  for each frame in propagate_in_video(start=key_z, reverse=True):
      result[frame] = logits > 0
```

> **Critical**: `apply_postprocessing=False` must be passed to `build_sam2_video_predictor_npz`. The default `fill_holes_in_mask_scores` CUDA kernel causes illegal memory access on some hardware.

#### `generate_bbox_review_cot(initial_bboxes, masks, needs_correction) → str`
Produces the Turn 5 review text listing PASS/FAIL with IoU for each annotated slice. Announces corrections for failing slices.

#### `generate_mask_review_cot(sampled, dice_per_z, needs_point) → str`
Produces the Turn 11 mask review text listing PASS/FAIL with Dice for each slice. Announces add_point corrections for failing slices.

#### `build_trajectory(anno, jpeg_root, render_dir, predictor, device) → dict | None`
Orchestrates the full 15-turn trajectory for one annotation:
1. Samples slices, generates spatial CoT
2. Jitters GT bboxes, optionally injects failures, identifies bbox corrections
3. Runs MedSAM2 propagation (or GT-derived fallback if no predictor)
4. Identifies slices needing point correction (Dice < 0.70)
5. Generates correction points from error regions
6. Renders all overlay images (bbox renders + mask renders)
7. Assembles messages list and images list in ms-swift format
8. Returns `None` if sample is unusable (too few lesion frames, no GT, etc.)

#### `validate_entry(entry) → (bool, str)`
Checks format correctness before writing to JSONL:
1. `<image>` token count == `images` list length
2. All `tool_call` JSON is valid and tool names are in the allowed set
3. bbox satisfies `x1 < x2`, `y1 < y2`
4. `add_point` has non-empty, matching-length `points` and `labels`
5. Last `tool_call` is `finish_3d_segmentation`
6. Every `tool_call` (except `finish_3d_segmentation`) has exactly one paired `tool_response`

### Hydra Config Path Fix

Absolute paths must be prefixed with `//` for Hydra:

```python
if os.path.isabs(cfg_path) and not cfg_path.startswith('//'):
    cfg_path = '//' + cfg_path
```

### Bbox Scaling

Bboxes are predicted in original image space and must be scaled to 512×512 before passing to MedSAM2:

```python
sx, sy = 512 / img_w, 512 / img_h
bbox_512 = np.array([x1*sx, y1*sy, x2*sx, y2*sy], dtype=np.float32)
```

## Training

### Environment

- **Conda env**: `qwen3` (ms-swift, transformers)
- **Base model**: `/BDSZ6/private/user/yxd/models/Qwen/Qwen3-VL-8B-Instruct`

### Key Training Parameters

| Parameter | Value | Rationale |
|---|---|---|
| `--agent_template hermes` | hermes | Required for Qwen3 parallel tool calling |
| `--max_length 12288` | 12288 | ~7,110 mean tokens; 8,192 filtered out 294/298 samples |
| `--tuner_type lora` | lora | LoRA rank 8, alpha 32, all-linear target |
| `--freeze_vit true` | true | Visual encoder frozen; only LLM layers trained |
| `--freeze_aligner true` | true | Aligner frozen for stability |
| `--packing true` | true | Sequence packing for efficient GPU utilisation |
| `--attn_impl flash_attn` | flash_attn | Flash Attention 2 for long sequences |
| `--deepspeed zero2` | zero2 | ZeRO-2 for multi-GPU (train_phase2.sh only) |

### Token Length Issue (Resolved)

**Problem**: With `max_length=8192`, only 4/298 training samples survived the length filter — all others exceeded the limit — resulting in only `num_train_epochs` total steps (1 step/epoch × N epochs) instead of the expected ~298.

**Root cause**: Each trajectory contained ~32 images. At ~333 tokens per 512×512 image, image tokens alone (~10,700) exceeded `max_length=8192` before any text was included.

**Fix applied**:
1. Render images (bbox overlays + mask overlays) resized to **256×256** (84 tokens each vs. 333)
2. `max_length` increased from 8,192 to **12,288**

After fix: mean total tokens = 7,110; all 298 samples fit within 12,288.

## JSONL Format (ms-swift hermes)

```jsonl
{
  "tools": "[{\"type\": \"function\", \"function\": {\"name\": \"add_bbox\", ...}}, ...]",
  "messages": [
    {"role": "user",         "content": "Task: ... Z=15: <image>\n...Z=57: <image>"},
    {"role": "assistant",    "content": "Global spatial analysis:\n..."},
    {"role": "tool_call",    "content": "{\"name\": \"add_bbox\", \"arguments\": {\"z_index\": 15, \"bbox\": [296, 342, 305, 355]}}"},
    ...
    {"role": "tool_response","content": "{\"z_index\": 15, \"bbox_image\": \"<image>\", \"iou_with_gt\": 0.89}"},
    ...
    {"role": "assistant",    "content": "Bbox review:\n  Z=15: PASS (IoU=0.89)\n..."},
    {"role": "tool_call",    "content": "{\"name\": \"run_medsam2\", \"arguments\": {\"key_z\": 38, \"bbox\": [145, 277, 355, 369]}}"},
    {"role": "tool_response","content": "{\"status\": \"propagation_complete\", \"key_z\": 38, \"slices\": [{\"z_index\": 15, \"mask_image\": \"<image>\", \"dice_with_gt\": 0.84}, ...]}"},
    {"role": "assistant",    "content": "MedSAM2 segmentation review:\n  Z=15: PASS (Dice=0.84)\n..."},
    {"role": "assistant",    "content": "All slices meet threshold. Segmentation complete. Finalising."},
    {"role": "tool_call",    "content": "{\"name\": \"finish_3d_segmentation\", \"arguments\": {}}"}
  ],
  "images": [
    "JPEGImages/case_001/00015.jpg",
    ...
    "agent_phase2/renders/case_001_z015_bbox.png",
    ...
    "agent_phase2/renders/case_001_z015_mask.png",
    ...
  ]
}
```

**Format invariants** (enforced by `validate_entry`):
- `<image>` token count == `images` list length (order must match across all messages)
- All tool calls are valid JSON with names in `{add_bbox, run_medsam2, add_point, finish_3d_segmentation}`
- Every `tool_call` except `finish_3d_segmentation` has exactly one paired `tool_response`
- Last `tool_call` is always `finish_3d_segmentation`
- bbox: `x1 < x2`, `y1 < y2`
